Scroll Score – Android Scroll Tracking App

An Android app that measures how much short-form content you scroll through, using Accessibility Services and per-app analytics.

Independent app project Kotlin • Android Studio Digital wellbeing • Analytics

Motivation & concept

Short-form content is addictive and hard to quantify. I wanted a way to see, in hard numbers, how much time I was sinking into endless scrolling on apps like TikTok, Instagram Reels, and YouTube Shorts.

Scroll Score is a background Android app that uses an Accessibility Service to listen for scroll events, counts them per app, and logs the results. The idea is simple: make it painfully obvious when “just a few minutes” has turned into hundreds of swipes.

Responsibilities

  • Designed the app architecture around AccessibilityService.
  • Implemented scroll event detection and debouncing logic.
  • Built the UI for per-app and daily history views.
  • Added CSV logging so data can be analysed externally.
Kotlin Android Accessibility WorkManager Data logging

Architecture & implementation

The core of Scroll Score is an AccessibilityService that listens for TYPE_VIEW_SCROLLED events from supported apps. Each event is tagged with the package name and time, then passed through a debouncer so a single “fling” doesn’t count as dozens of scrolls.

Counts are stored in a local database and aggregated by:

  • Current session (since app opened or reset).
  • Per-app daily totals.
  • Rolling history, exportable as CSV.

A WorkManager job runs once per day to roll over stats and optionally write a CSV snapshot. The UI is intentionally simple: one main screen with today’s total, app breakdown, and a plain history list.

Outcomes & key learnings

Scroll Score works reliably across major social apps and showed me just how quickly “a few scrolls” adds up. It also forced me to learn some of the more unusual corners of Android development, like Accessibility events and background work scheduling.

Technically, this project helped me get comfortable with:

  • Android’s event system and how accessibility ties into the UI tree.
  • Designing debouncing logic that balances accuracy vs. noise.
  • Persisting and exporting data for analysis outside the app.
